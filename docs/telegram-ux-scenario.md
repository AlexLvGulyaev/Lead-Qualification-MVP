# Telegram UX MVP - Сценарий

## Обзор

**Тип:** Lead Capture Channel (не AI-консультант, не чат-бот поддержки)

**Назначение:** Сбор заявки → Передача в LQ → Подтверждение регистрации

---

## Главное меню

### Команда /start

**Сообщение:**
```
Здравствуйте!

Вы можете оставить заявку для связи с менеджером.

Выберите действие.
```

**Клавиатура (ReplyKeyboardMarkup):**
```
[Создать заявку] [Помощь]
[О проекте]
```

---

## Сценарий "Создать заявку"

### Шаг 1: Имя

**После нажатия "Создать заявку":**

1. Обновить сессию: `step = 'name'`
2. Отправить сообщение:
```
Как вас зовут?
```

**Обработка ответа:**
1. Сохранить `collected_data.name`
2. Обновить сессию: `step = 'phone'`
3. Перейти к шагу 2

---

### Шаг 2: Телефон

**Сообщение:**
```
Укажите номер телефона для связи.
```

**Обработка ответа:**
1. Сохранить `collected_data.phone`
2. Обновить сессию: `step = 'email'`
3. Перейти к шагу 3

---

### Шаг 3: E-mail

**Сообщение:**
```
Укажите ваш e-mail (необязательно).
```

**Клавиатура (ReplyKeyboardMarkup):**
```
[Пропустить]
```

**Обработка ответа:**
- Если "Пропустить": `collected_data.email = null`
- Иначе: `collected_data.email = text`
1. Обновить сессию: `step = 'description'`
2. Перейти к шагу 4

---

### Шаг 4: Описание

**Сообщение:**
```
Кратко опишите ваш вопрос или задачу.
```

**Обработка ответа:**
1. Сохранить `collected_data.description`
2. Обновить сессию: `step = 'confirm'`
3. Перейти к шагу 5

---

### Шаг 5: Подтверждение

**Сообщение:**
```
Проверьте данные заявки.

Имя: {{name}}
Телефон: {{phone}}
E-mail: {{email}}
Описание: {{description}}

Всё верно?
```

**Клавиатура (InlineKeyboardMarkup):**
```
[✅ Отправить] [✏️ Изменить] [❌ Отмена]
```

**Callback Data:**
- `confirm_yes` — отправить заявку
- `confirm_edit` — изменить данные
- `confirm_cancel` — отменить

---

### После подтверждения (confirm_yes)

1. Создать контакт (find_or_create_contact_by_telegram)
2. Создать лид (INSERT INTO leads)
3. Создать сообщение (INSERT INTO messages)
4. Записать лог (INSERT INTO logs)
5. Отправить подтверждение:

**Сообщение:**
```
Спасибо!

Ваша заявка зарегистрирована.

Номер: LQ-XXXXXX

Мы передали информацию в обработку.
```

6. Обновить сессию: `step = 'done'`, `completed_at = NOW()`

---

### При выборе "Изменить" (confirm_edit)

1. Обновить сессию: `step = 'name'`
2. Отправить сообщение:
```
Давайте начнём сначала.

Как вас зовут?
```

---

### При выборе "Отмена" (confirm_cancel)

1. Обновить сессию: `step = 'start'`, `collected_data = '{}'`
2. Отправить сообщение:
```
Заявка отменена.

Вы можете начать заново через меню.
```

## Обработка ошибок

### Пользователь прервал сценарий

Если пользователь отправил `/start` в середине сценария:
1. Сбросить сессию: `step = 'start'`
2. Показать главное меню

### Пустое обязательное поле

Если пользователь не заполнил обязательное поле:
1. Повторить запрос с сообщением:
```
Это поле обязательно для заполнения.

{{текущий запрос}}
```

### Слишком короткое описание

Если описание < 10 символов:
```
Описание слишком короткое. Минимум 10 символов.

Кратко опишите ваш вопрос или задачу.
```

## Хранение состояния

**Таблица:** `telegram_sessions`

| Поле | Тип | Описание |
|------|-----|----------|
| telegram_id | VARCHAR(255) | ID пользователя в Telegram |
| chat_id | BIGINT | ID чата |
| current_step | VARCHAR(50) | Текущий шаг: start, name, phone, email, description, confirm, done |
| collected_data | JSONB | Собранные данные: {name, phone, email, description} |
| created_at | TIMESTAMP | Время создания |
| updated_at | TIMESTAMP | Время обновления |
| completed_at | TIMESTAMP | Время завершения |

---

## Функции PostgreSQL

```sql
-- Получить или создать сессию
get_or_create_telegram_session(telegram_id, chat_id, username, first_name, last_name)

-- Обновить шаг и данные
update_telegram_session_step(telegram_id, step, data_key, data_value)

-- Завершить сессию
complete_telegram_session(telegram_id)

-- Сбросить сессию
reset_telegram_session(telegram_id)
```

## Telegram API

### Отправка сообщения с клавиатурой

**ReplyKeyboardMarkup (главное меню):**
```json
{
  "chat_id": "{{chat_id}}",
  "text": "Здравствуйте! ...",
  "reply_markup": {
    "keyboard": [
      [{"text": "Создать заявку"}, {"text": "Помощь"}],
      [{"text": "О проекте"}]
    ],
    "resize_keyboard": true,
    "one_time_keyboard": false
  }
}
```

**InlineKeyboardMarkup (подтверждение):**
```json
{
  "chat_id": "{{chat_id}}",
  "text": "Проверьте данные...",
  "reply_markup": {
    "inline_keyboard": [
      [
        {"text": "✅ Отправить", "callback_data": "confirm_yes"},
        {"text": "✏️ Изменить", "callback_data": "confirm_edit"}
      ],
      [
        {"text": "❌ Отмена", "callback_data": "confirm_cancel"}
      ]
    ]
  }
}
```

### Ответ на callback query

```json
{
  "callback_query_id": "{{callback_query_id}}",
  "text": "Заявка отправлена",
  "show_alert": false
}
```

## Требования к n8n Workflow

1. **Telegram Trigger** — принимает сообщения и callback queries
2. **Parse Update** — парсит входящие данные
3. **Get Session** — получает или создаёт сессию из PostgreSQL
4. **Route Request** — определяет маршрут по шагу и типу сообщения
5. **Switch** — ветвление по маршрутам
6. **Nodes for each step** — обработка каждого шага
7. **Send Message** — отправка сообщений с клавиатурами
8. **Create Lead** — создание лида после подтверждения

---

## Ограничения

1. Нет свободного диалога с LLM
2. Нет RAG и консультаций
3. Нет длинных сценариев
4. Сессия хранится в PostgreSQL (не в Redis)
5. Нет уведомлений менеджерам через Telegram
