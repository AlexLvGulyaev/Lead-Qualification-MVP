# Инструкция по настройке полей в Kommo CRM

**Для проекта:** Lead Qualification Assistant  
**Цель:** Создать кастомные поля для синхронизации лидов  
**Аудитория:** Пользователь без опыта работы с Kommo

---

## Обзор

Эта инструкция поможет создать в Kommo CRM поля, необходимые для автоматической синхронизации лидов из Lead Qualification Assistant.

**Что будет создано:**
- 4 обязательных поля
- 2 опциональных поля

**Время выполнения:** ~15 минут

---

## Предварительные требования

✅ Kommo аккаунт создан  
✅ Pipeline и статусы существуют  
✅ API-доступ проверен

## Шаг 1. Открытие раздела настроек полей

### 1.1. Перейти в раздел «Сделки»

1. В левом меню нажмите на **«Сделки»** (Leads)

### 1.2. Открыть настройку полей

1. Кликните на **любую сделку** в списке (или создайте новую)
2. В открывшейся карточке сделки **слева** найдите вкладку **«Setup»** (Настройка)
3. Нажмите на **«Setup»**
4. Откроется шаблон для создания полей

**Альтернативный путь:** В разделе «Сделки» → значок шестерёнки → Поля

### 1.3. Выбрать раздел для полей

В открывшемся шаблоне выберите раздел:
- **Lead** — для полей сделки (нам нужен этот раздел)

## Шаг 2. Создание обязательных полей

### Поле 1: Lead Type (Тип лида) — ОБЯЗАТЕЛЬНО

#### Что это такое
Это поле определяет, насколько «горячий» лид. Значение проставляется автоматически AI-классификатором.

#### Значения
- `hot` — готов купить прямо сейчас
- `warm` — заинтересован, нужен follow-up
- `cold` — не готов к решению
- `spam` — нецелевое обращение

#### Инструкция

1. В разделе **Lead** нажмите **«+ Add field»** (Добавить поле)

2. В открывшемся окне заполните:
   
   | Поле | Значение |
   |------|----------|
   | **Name** (Название) | `Lead Type` |
   | **Type** (Тип) | **Select** (Выпадающий список) |

3. Добавьте варианты (enums):
   
   Нажмите «Add option» или аналогичную кнопку и введите значения по одному:
   
   | Value | Sort |
   |-------|------|
   | `hot` | 1 |
   | `warm` | 2 |
   | `cold` | 3 |
   | `spam` | 4 |

4. Нажмите **«Save»** (Сохранить)

5. **Запишите ID поля:**
   
   После сохранения поле появится в списке. Нажмите на поле, чтобы увидеть его **ID** (field_id).
   
   Запишите: `KOMMO_LEAD_TYPE_FIELD_ID = ________`

**Зачем нужно:** Автоматическое распределение лидов по статусам. hot → срочный звонок, warm → email, cold → архив.

---

### Поле 2: Priority (Приоритет) — ОБЯЗАТЕЛЬНО

#### Что это такое
Приоритет обработки лида. Помогает менеджерам сортировать очередь работы.

#### Значения
- `high` — высокий приоритет
- `medium` — средний приоритет
- `low` — низкий приоритет

#### Инструкция

1. Нажмите **«+ Add field»**

2. Заполните:
   
   | Поле | Значение |
   |------|----------|
   | **Name** | `Priority` |
   | **Type** | **Select** |

3. Добавьте варианты:
   
   | Value | Sort |
   |-------|------|
   | `high` | 1 |
   | `medium` | 2 |
   | `low` | 3 |

4. Нажмите **«Save»**

5. **Запишите ID поля:**
   
   `KOMMO_PRIORITY_FIELD_ID = ________`

**Зачем нужно:** Помогает менеджерам понять, какой лид важнее обрабатывать первым.

---

### Поле 3: Confidence (Уверенность) — ОБЯЗАТЕЛЬНО

#### Что это такое
Насколько уверен AI в своей классификации. Число от 0.00 до 1.00.

#### Значения
- Минимум: `0`
- Максимум: `1`
- Примеры: `0.92`, `0.85`, `0.55`

#### Инструкция

1. Нажмите **«+ Add field»**

2. Заполните:
   
   | Поле | Значение |
   |------|----------|
   | **Name** | `Confidence` |
   | **Type** | **Numeric** (Числовое) |

3. Нажмите **«Save»**

4. **Запишите ID поля:**
   
   `KOMMO_CONFIDENCE_FIELD_ID = ________`

**Зачем нужно:** Показывает надёжность классификации. Если confidence низкий (ниже 0.50), используется fallback-классификация.

**Важно:** Numeric-поля в Kommo принимают только **неотрицательные** числа.

---

### Поле 4: Public Number (Номер обращения) — ОБЯЗАТЕЛЬНО

#### Что это такое
Человекочитаемый номер обращения в формате LQ-NNNNNN. Например: LQ-123456.

#### Значения
- Формат: `LQ-` + 6 цифр
- Примеры: `LQ-000001`, `LQ-123456`, `LQ-999999`

#### Инструкция

1. Нажмите **«+ Add field»**

2. Заполните:
   
   | Поле | Значение |
   |------|----------|
   | **Name** | `Public Number` |
   | **Type** | **Text** (Текстовое) |

3. Нажмите **«Save»**

4. **Запишите ID поля:**
   
   `KOMMO_PUBLIC_NUMBER_FIELD_ID = ________`

**Зачем нужно:** Позволяет клиенту и менеджеру быстро находить обращение по номеру. Удобнее, чем UUID.

## Шаг 4. Получение ID полей через API

Если ID не отображается в интерфейсе, используйте API.

### Команда для выполнения на VPS

```bash
curl -s -X GET \

  -H "Authorization: Bearer ${KOMMO_ACCESS_TOKEN}" \
  -H "Content-Type: application/json" | jq '.'
```

### Как найти нужный ID

Ищите поле по `name` и берите значение `id`:

```bash
# Найти ID поля "Lead Type"
curl -s -X GET \
  "${KOMMO_BASE_URL}/api/v4/leads/custom_fields" \
  -H "Authorization: Bearer ${KOMMO_ACCESS_TOKEN}" \
  -H "Content-Type: application/json" | \
  jq '._embedded.custom_fields[] | select(.name == "Lead Type") | .id'
```

**Структура ответа:**

API возвращает JSON с обёрткой `_embedded.custom_fields`. Каждое поле содержит:
- `id` — уникальный идентификатор поля
- `name` — название поля
- `type` — тип поля (`text`, `numeric`, `select`, `multiselect`, `radiobutton`, и др.)
- `sort` — порядок сортировки
- `enums` — массив вариантов для полей типа `select`/`multiselect`/`radiobutton` (каждый enum имеет `id`, `value`, `sort`)

**Важно:** ID полей и enums генерируются автоматически при создании поля.

## Важные замечания

### Тип поля нельзя изменить

После создания поля изменить его тип **невозможно**. Если ошиблись:
1. Удалите поле
2. Создайте заново с правильным типом
3. Данные удалятся

### Numeric — только неотрицательные числа

Поле типа **Numeric** принимает только числа ≥ 0. Для отрицательных чисел используйте **Text**.

### Только администраторы могут создавать поля

Убедитесь, что ваш аккаунт имеет права администратора.

## Частые вопросы

### Q: Где находится раздел «Сделки»?

**A:** В левом меню Kommo. Если не видите — проверьте права доступа.

### Q: Где увидеть ID поля?

**A:** Два способа:
1. Нажмите на поле в списке — ID отобразится в карточке поля
2. Используйте API-запрос из Шага 4

### Q: Можно ли использовать русские названия?

**A:** Да, но для API лучше использовать английские названия (field_name). Значения в Select могут быть на любом языке.

### Q: Что делать, если создал поле с ошибкой?

**A:** Удалите поле и создайте заново. Данные в этом поле удалятся.

## Альтернатива: Создание полей через API

Если вы предпочитаете автоматизировать создание полей, используйте API.

### Создание Select-поля через API

```bash
curl -X POST \
  "${KOMMO_BASE_URL}/api/v4/leads/custom_fields" \
  -H "Authorization: Bearer ${KOMMO_ACCESS_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "name": "Lead Type",
      "type": "select",
      "sort": 500,
      "enums": [
        {"value": "hot", "sort": 1},
        {"value": "warm", "sort": 2},
        {"value": "cold", "sort": 3},
        {"value": "spam", "sort": 4}
      ]
    }
  ]'
```

### Создание Numeric-поля через API

```bash
curl -X POST \
  "${KOMMO_BASE_URL}/api/v4/leads/custom_fields" \
  -H "Authorization: Bearer ${KOMMO_ACCESS_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "name": "Confidence",
      "type": "numeric",
      "sort": 520
    }
  ]'
```

### Создание Text-поля через API

```bash
curl -X POST \
  "${KOMMO_BASE_URL}/api/v4/leads/custom_fields" \
  -H "Authorization: Bearer ${KOMMO_ACCESS_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "name": "Public Number",
      "type": "text",
      "sort": 530
    }
  ]'
```

### Структура запроса для Select-поля

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `name` | string | ✅ Да | Название поля |
| `type` | string | ✅ Да | Тип: `text`, `numeric`, `select`, `multiselect`, `radiobutton`, и др. |
| `sort` | integer | ❌ Нет | Порядок сортировки (по умолчанию 500) |
| `enums` | array | ✅ Для select/multiselect/radiobutton | Массив вариантов выбора |
| `enums[].value` | string | ✅ Да | Текстовое значение |
| `enums[].sort` | integer | ✅ Да | Порядок сортировки |
| `enums[].code` | string | ❌ Нет | Символьный код (для обновлений без ID) |

**Важно:** После создания поля тип изменить невозможно. При ошибке — удалите поле и создайте заново.
